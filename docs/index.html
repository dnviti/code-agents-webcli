Run code agents from the browser

A single Node.js app with GitHub OAuth, SQLite persistence, multi-user isolation, and terminal sessions powered by xterm.js.

npx code-agents-webcli --setup

🔐 GitHub OAuth enabled
🧠 Claude / Codex / Terminal sessions available

Features

GitHub OAuth

Users authenticate with GitHub and are identified internally by GitHub numeric ID.

SQLite Persistence

Runtime sessions, auth sessions, and user settings survive restarts in a local SQLite database.

Multi-user Isolation

Sessions are owner-scoped so users only see and reconnect to their own work.

xterm.js Frontend

Modern terminal rendering with theme and font controls from the browser.

Docker Ready

The repo ships with a Dockerfile and GitHub Actions workflow for container builds.

npm + GHCR Release

Release automation publishes the package to npm and the image to GitHub Container Registry.

Get Started

Run locally

npm install -g code-agents-webcli
code-agents-webcli --setup

# or without installing
npx code-agents-webcli --setup

Configure OAuth callback

https://your-host.example.com/auth/github/callback

# local development
http://localhost:32352/auth/github/callback

Deploy

  • Database: local state is stored in ~/.code-agents-webcli/app.sqlite unless overridden with --data-dir.
  • Docker: the container bundles the web app, but you still need Claude / Codex / Cursor CLIs in the runtime environment.
  • Publishing: npm trusted publishing must be configured on npm before the GitHub release workflow can publish successfully.
